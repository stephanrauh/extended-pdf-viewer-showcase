The [readingDirection] attribute controls the reading direction independently from the UI language.

  • 'auto' (default): inherits the direction from the UI language. If the UI is set to Arabic or Hebrew, pages automatically display right-to-left. In most cases, this is all you need.
  • 'ltr': forces left-to-right page ordering, even with an RTL UI language.
  • 'rtl': forces right-to-left page ordering when the UI language is LTR (e.g., English). Use this when displaying Arabic, Hebrew, or Farsi documents in an English UI. Also useful for certain Japanese books that read right-to-left.

In RTL mode:

  • Spread pages are displayed right-to-left (right page first)
  • Arrow key navigation is reversed (left arrow = next page, right arrow = previous page)
  • Horizontal and wrapped scroll modes display pages right-to-left

Try enabling a spread mode and switching between LTR and RTL to see the page order change. Use the left/right arrow keys to verify that navigation direction is also reversed.
