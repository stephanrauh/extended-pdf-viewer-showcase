ngx-extended-pdf-viewer ships with two versions of pdf.js:
  • The assets folder contains the latest release of pdf.js.
  • The bleeding-edge folder contains a developer build of pdf.js. That code is newer than the latest release, but it's not an officially supported version, so it's primarily there for testing purposes.
You can switch to the newer either by modifying the angular.json or by setting pdfDefaultOptions.assetsFolder.
This option also allows to use use a non-standard project directory layout.
Note that it's not possible to change the pdf.js engine at runtime. You need to set it before <ngx-extended-pdf-viewer/> is loaded. This demo uses the "bleeding edge" version of pdf.js after a full-page reload (i.e. after hitting the F5 key).
