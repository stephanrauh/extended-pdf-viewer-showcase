Attributes, events, and attributes with two-way-binding

Legend:

  • [(attribute)] describes an attribute with two-way-binding
  • [attribute] means that PDF-viewer reacts when the attribute changes
  • (attribute) means an event is raised when the user changes a setting
  • attribute (without special characters) means the attribute is used at load time only. Subsequent changes are ignored.

Accessing the base library directly

The PDF viewer is based on a Mozilla project, pdf.js, which offers many options and features I didn't expose as an API of ngx-extended-pdf-viewer. Nonetheless, it's there, so you can use it as long as you know what you're doing. The API I'm describing below isn't likely to change any time soon, but that's difficult to predict, so be careful. If you run into such a change, please open an issue at GitHub so I can update the documentation. Note that I won't try to keep my fork of the base library compatible. That's simply impossible with such an active project.
