Starting with version 16.2.14, ngx-extended-pdf-viewer supports server-side rendering out-of-the-box.

Since version 21.3.4, the showcase uses static site generation (SSG). That doesn't mean the PDF files are generated on the server, but it means you can use ngx-extended-pdf-viewer in a server-side rendered Angular application. If there's a bug, it'll probably show up in the showcase, allowing me to detect and fix it.

Note: Versions 22 and 23 of the showcase temporarily disable SSG. That's because of updating the showcase to Angular 19. That didn't work smoothly. Both server-side rendering and SSG work great in Angular 19, it's just that it broke when I ran the automated updates. I assume that this is just my unwillingness to read the manual. :) I'll fix this ASAP. (All right, ASAP is stretching it a bit - at the moment, I'm running short of leisure time, so it's slow going. Bear with me.)

Before version 16.2, ngx-extended-pdf-viewer used a special module to support server-side rendering. If you're still using it, update to ngx-extended-pdf-viewer 17+ and abandon NgxExtededPdfViewerServerModule in favor of the new, unified approach.
