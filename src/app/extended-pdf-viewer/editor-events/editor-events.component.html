Use case: when the user is drawing or highlighting text, the editor options block a part of the PDF file. So how can we hide the editor options while drawing? Editor events to the rescue!

The drawingStarted and drawingStopped events fire when the user starts and stops actively drawing or highlighting. Use them to hide the editor options popup (color, thickness, opacity) while drawing. Each editor has its own toolbar with a unique ID (e.g. editorInkParamsToolbar, editorHighlightParamsToolbar). Hide the toolbar by adding the hidden CSS class.

A frequent question (see issue #3228): "After I resized an annotation I want a screenshot of that area, but the x / y / width / height are strange numbers below 1."

Those numbers are normalized coordinates: fractions between 0 and 1, relative to the page, measured from the top-left corner. They are not pixels and not PDF points. See the dedicated Coordinate systems page for the full story.

Add a highlight, draw, or image annotation in the viewer below (and resize or move it). The viewer keeps reporting the editor's normalized rectangle. Because getPageAsCanvas() accepts a cropBox in exactly these coordinates, screenshotting the annotation needs no coordinate maths at all.
